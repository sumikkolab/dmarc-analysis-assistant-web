# DMARC 分析支援ツール

**DMARC レポートを、対応すべきものが見える形で可視化。**

[English README](README.md)

## このツールについて

DMARC 分析支援ツールは、DMARC 集約レポート（RUA）と関連するバウンスメールを **ローカル端末のみ** で解析し、6 分類で判定して、対応すべきものと放置してよいものを区別する Windows デスクトップアプリです。

クラウドサービスではありません。レポート本文・IP 情報・認証結果・ドメイン情報を外部に送信することは一切ありません。

## なぜこのツールが必要か

DMARC 集約レポートは XML で届き、送信元 IP・SPF/DKIM 結果・件数が並ぶだけで、パッと見ても次のどれに該当するか分かりません：

- 対応すべき **なりすまし疑い**
- 自社送信経路の **設定不備**
- 放置してよい **転送由来**

本ツールは、決定論的なルールベース分類で上記を区別し、運用者が本当に対応すべきものに時間を使えるよう支援します。

## このツールでできること

### ワークフロー: 取込 → 解析 → 分類 → 確認 → エクスポート

| ステップ | 内容 |
|---------|------|
| **取込** | フォルダから DMARC 集約レポート（XML / ZIP / GZ）を読み込み、または Microsoft 365 受信トレイから取得（任意） |
| **解析** | 送信元 IP、認証結果（SPF / DKIM）、件数、ポリシー評価、ヘッダーメタ情報を抽出 |
| **分類** | 6 分類のルールベースエンジンで各エントリを判定 |
| **確認** | フィルタ付き一覧と、ドメイン別・分類別・期間別のトレンドを表示 |
| **エクスポート** | 分類結果を CSV で出力、Excel / BI ツールで追加分析が可能 |

### 6 分類ラベル

| ラベル | 意味 |
|--------|------|
| **なりすまし疑い** | SPF / DKIM 両方 fail、外部 IP 起点。最優先対応。 |
| **設定不備** | 自社送信のはずが認証失敗。DNS / 送信経路の確認が必要。 |
| **転送由来** | 転送が原因の fail。通常は対応不要。 |
| **正常送信** | pass。記録として保持。 |
| **バウンス** | バウンスメールと突合した結果。 |
| **判定不能** | 情報不足で自動判定できなかったもの。 |

### 主な機能

- ZIP / GZ / XML 各形式に対応した DMARC 集約レポート解析
- バウンスメール突合による転送由来 / 実問題の切り分け
- ドメイン別・分類別・期間別のトレンド集計
- CSV エクスポート
- 日本語 / 英語 完全対応
- **データはすべてローカル保存** — 外部サーバーへの送信なし
- 決定論的ルールベース分類 — AI 推論・クラウド解析は使用しません

## プライバシーとセキュリティ

- 本ツールはローカル端末上でのみ動作します
- レポート本文・IP アドレス・認証結果・ドメイン情報を外部に **送信しません**
- Microsoft 365 連携は **任意** — 利用者が明示的に設定した場合のみ有効
- 権限スコープは `User.Read` + `Mail.Read` のみ（読み取り専用 — 送信・削除・編集なし）
- 認証トークンはメモリ上にのみ保持、ディスクには保存しません
- テレメトリ・クラッシュレポート・使用履歴の自動送信 **なし**
- 保存データは `%LocalAppData%\DmarcAnalysisAssistant` 配下、利用者の操作でいつでも削除可能
- 初回起動時に同意が必要。規約更新時に再同意

## 動作環境

- Windows 10 / 11
- .NET 8 Desktop Runtime
- Microsoft Edge WebView2 Runtime
- Microsoft 365 Graph API アクセス（任意のメール取込機能を使う場合のみ）
